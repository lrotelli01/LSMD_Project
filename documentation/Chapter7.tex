\chapter{API Documentation \& Testing}
    \section{Backend Interface: Swagger UI}

	For our backend documentation, we integrated Swagger UI. This tool provides a standardized interface to visualize all our REST endpoints. As you can see, the API is organized by Controllers—Guest, Customer, and Manager—mirroring the actors in our system. It allows us to interactively test the backend logic independently from the client.
    The Swagger UI is accessible at the following URL:
\begin{center}
    \url{http://localhost:8080/swagger-ui/index.html\#/}
\end{center}
    \vspace{0.5cm}
    
    \begin{itemize}
        \item \textbf{Visualization:} Provides a clear, interactive view of all RESTful endpoints.
        \item \textbf{Structure:} The API is organized by Controllers:
        \begin{itemize}
            \item \texttt{Messaging}
            \item \texttt{Manager Dashboard}
            \item \texttt{Authentication}
            \item \texttt{Payment Methods}
            \item \texttt{Properties}
            \item \texttt{Favorites}
            \item \texttt{Reservations}
            \item \texttt{User Account}
            \item \texttt{Notifications}
        \end{itemize}
        \item \textbf{Independence:} Allows testing server logic and database queries directly, without relying on the frontend client.
    \end{itemize}
